VulsRepo v0.2.5 supports Affected Processes and Trivy information

November 17, 2020

2 minutes read

 Pinterest

VulsRepo online demo v.0.2.5 is here.

v0.2.5

Features

  • #105 Affected Processes information is now viewable.

  • #143 Trivy information is now viewable. Added Path and Title on pivot table and references on detail panel.

  • #151 You can now choose to show or hide CveCcontents.

  • #151 Changed to sort the CVSS Detail according to priority.
  • #144 Changed CVSS Vector chart to distinguish between CVSS v3.0 and CVSS v3.1.
  • #144 When the priority of the JVN is higher than the NVD, The items in the CVSS Vector Chart have been translated into Japanese.

  • #137 Added version display.

Changes

  • #144 Changed to omit non-existent data from CVSS Vector chart.
  • #151 Priority is no longer reset to the initial value when upgrading.
  • #162 Changed AdvisoryID width.
  • #105 Changed CweID width.
  • #159 Changed documentation url.

Fixes

  • #165 Fixed problem where data could not be read and stopped at Please Wait... when warnings were null instead of [].
  • #142 Fixed Red Hat Network link doesn’t work on pivot table.
  • #160 Fixed Oracle Linux Errata link doesn’t work on pivot table.
  • #147 Fixed wrong CVSS Severity on detail panel.
  • #153 Fixed an issue where CweID rank badges are displayed when no information is available.
  • #153 If CveContents does not have CweID information, CweID information is now retrieved from the next CveContents.
  • #160 If cvss3Severity does not exist and distroAdvisories.severity exists, distroAdvisories.severity is now shown as CVSS Severity. (Oracle Linux 8 only?)
  • #162 Fixed an issue where only one AdvisoryID was displayed when there were multiple AdvisoryID.

Docker image

ishidaco/vulsrepo - Docker Hub 

docker pull ishidaco/vulsrepo

docker pull ishidaco/vulsrepo:version-0.2.5

  • Oracle Linux 7.9
  • Alpine Linux on Docker container
